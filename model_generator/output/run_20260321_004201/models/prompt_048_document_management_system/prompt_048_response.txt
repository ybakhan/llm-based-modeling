CONSTRUCTS_INVOLVED: Upload Document, Validate File Format, the <<include>> relationship from Upload Document to Validate File Format
EXPLANATION: "Validate File Format" is an inclusion use case that is included by only one use case ("Upload Document"). It is not associated with any actor directly. It does not include or extend any other use case, is not extended by any use case, and is not involved in any generalization. "Validate File Format" represents a sub-step of uploading a document rather than a complete, independently meaningful service observable to an actor. This is a classic functional decomposition — breaking the "Upload Document" use case into an artificial sub-function via <<include>>.

REFACTORING_RATIONALE: The "Validate File Format" use case and its <<include>> relationship from "Upload Document" must be removed. The validation behavior should be absorbed into "Upload Document" itself, since file format validation is an internal step of uploading a document, not a standalone service. This eliminates the functional decomposition by merging the decomposed sub-function back into its base use case. No new use cases are introduced; we simply remove "Validate File Format" and the include arrow.
