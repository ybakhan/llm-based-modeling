CONSTRUCTS_INVOLVED: Submit Application (UC2), Upload Transcript (UC3), include relationship from UC2 to UC3
EXPLANATION: "Upload Transcript" (UC3) is an inclusion use case that is included by only one use case ("Submit Application" / UC2). It is not associated with any actor directly, does not include or extend any other use case, is not extended by any other use case, and is not involved in any generalization. "Upload Transcript" does not represent a complete, independently valuable service to any actor — it is merely a procedural step of submitting an application that has been improperly decomposed into a separate use case. This is a textbook instance of Functional Decomposition via the include relationship.

REFACTORING_RATIONALE: The inclusion use case "Upload Transcript" (UC3) and its <<include>> relationship from "Submit Application" (UC2) must be removed. The transcript upload behavior should be merged into the "Submit Application" use case, which already represents the complete service of applying for a scholarship. This eliminates the unnecessary functional decomposition, reducing the model by one use case and one include relationship (net −2 constructs).
