This constitutes the Functional Decomposition antipattern via the include relationship.

REFACTORING_RATIONALE: The inclusion use case "Select Time Slot" (UC4) must be merged back into its sole base use case "Book Service" (UC2). This reduces the model by one use case and one include relationship, yielding a cleaner model where "Book Service" encapsulates the full booking behavior including time slot selection.
